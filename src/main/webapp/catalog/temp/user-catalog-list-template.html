Java JenSoftAPI Overview

 JenSoftAPI™ is a pure java framework to make java charts components. API provides a complete charting solution for your Swing or Web Application. It provides advanced features you've come to expect with minimal integration efforts. Framework gives developers a very flexible toolkit solution for a wide variety of java chart and data visualization needs. The API can be used in many businesses such as reporting, health, finance or science and technology softwares.

 Framework provides data oriented plug-ins for most of commons charts components: Curve or Line, Plot or Scatter, Area, Stacked Area, Bar Symbol, Stacked Bar Symbol, Point Symbol,Stock, Pie, Pie 3D, Donut2D, Scan, Legend, Cloud points, Real time.You can easily extend and override components to meet your needs.

 Framework provides tool oriented plug-ins for most of commons transforms and user/chart interaction: Zoom Box, Zoom Lens, Zoom Wheel, Zoom Percent, Translate and Marker. This plug-ins generally offers nice transition during process operations and nice widgets and popup context to lock tool, launch commands or other user intents.

 Deploy Desktop or Server Apps : use in desktop application and benefits off all swing chart API with very rich interaction and high level user and developer control on Desktop standalone app or cllient web start or applets. Deploy in a web container and process charts images through network according to framework server components.

 Driven development by online catalog samples : Online catalog reflects all catalog samples artifact in a sleek and modern web module. Browse more than 250 samples charts with demo and source code.

Getting Started

The API is mainly based on 3 components : View defines the end chart, Window which is the user projection and plugins that take the responsibility of painting process and handle user intents. In Web server, this 3 components will be sufficient to produce chart. In swing context, you can use widget and context menu for user/chart interaction.
Plugin is everything : By plugins composition, you can achieve rich charts by multiple layer of projection(s) and/or plugin(s).You can also implement your own plugin to add new feature. 

						
// 1 view2d is the end user visible chart component.
View2D view2D  = new View2D();

// 2 window is a user projection that have be to register in view
Window2D w2d = new Window2D.Linear(-10, 12, -5, 26);
view2D.registerWindow2D(w2d);

// 3 plug-ins paint a scene and handle user interaction in a projection
FooPlugin fooPlugin = new FooPlugin();
BarPlugin barPlugin = new BarPlugin();

// 4 plug-ins are registered in a window user projection.
w2d.registerPlugin(fooPlugin);
w2d.registerPlugin(barPlugin);

// 5 plug-ins may have sometimes widgets
FooWidget fwidget = new FooWidget();
fooPlugin.registerWidget(fwidget);
	
// 6 plug-ins may have sometimes context menu
BarContext bcontext = new BarContext();
barPlugin.registerContext(bcontext);
